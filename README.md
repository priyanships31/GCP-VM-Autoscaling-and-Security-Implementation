# GCP VM Autoscaling and Security Implementation

## 📌 Project Overview

This project demonstrates the deployment of a scalable and secure Virtual Machine infrastructure on **Google Cloud Platform (GCP)**.

The implementation includes:

- Standalone Virtual Machine creation
- Instance Template configuration
- Managed Instance Group (MIG)
- CPU-based Autoscaling
- Firewall rule configuration
- IAM custom role for restricted access
- Deployment automation using gcloud CLI

The solution ensures scalability, security, and reproducibility.

---

## 🏗 Architecture Components

### 1️⃣ Standalone VM

- **Name:** `web-server-vm`
- **Machine Type:** e2-medium
- **OS:** Debian 12
- Used to demonstrate manual VM provisioning.

---

### 2️⃣ Instance Template

- **Name:** `compute-template-v1`
- **Machine Type:** e2-medium
- **Boot Disk:** Debian 12
- Defines configuration used by the Managed Instance Group.

The template ensures all auto-created instances are identical.

---

### 3️⃣ Managed Instance Group (MIG)

- **Name:** `gcp-web-mig`
- **Region:** us-central1
- **Minimum Instances:** 1
- **Maximum Instances:** 10
- **Base Instance Name:** gcp-web-mig

The MIG automatically creates and manages VM instances using the instance template.

### Auto-Created VM Instance

The MIG automatically created the following VM:

- `gcp-web-mig-g7mh`

Instance naming format:

gcp-web-mig-<random-suffix>



The suffix is automatically generated by GCP.

This proves that the instance was created dynamically by the Managed Instance Group.

---

## 🔄 Autoscaling Configuration

Autoscaling is configured based on CPU utilization.

### Policy Settings:

- **Metric:** CPU utilization
- **Target CPU:** 60%
- **Minimum Instances:** 1
- **Maximum Instances:** 10
- **Cooldown Period:** 60 seconds

### How Autoscaling Works

- If average CPU > 60% → New VM instance is created.
- If average CPU < 60% → Extra instances are removed.
- The system always maintains at least 1 instance.

The autoscaler monitors the Managed Instance Group and dynamically adjusts capacity based on workload.

---

## 🔐 Security Implementation

### Firewall Rules

Configured under VPC Network:

1. **allow-http**
   - Direction: Ingress
   - Protocol: TCP
   - Port: 80
   - Source: 0.0.0.0/0

2. **allow-ssh-restricted**
   - Direction: Ingress
   - Protocol: TCP
   - Port: 22
   - Source: Specific public IP only
   - Target Tag: ssh-access

This ensures controlled and secure access.

---

### IAM Configuration

A custom IAM role was created:

- **Role Name:** ComputeViewerRestricted
- **Permissions:**
  - compute.instances.get
  - compute.instances.list

Assigned to:

- **Service Account:** `restricted-sa@gcp-vm-autoscaling-security.iam.gserviceaccount.com`

This follows the Principle of Least Privilege by restricting access to read-only VM operations.
